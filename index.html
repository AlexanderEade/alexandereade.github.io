GitHub Explanation

When creating this website, we started by creating a Github repository to host our HTML files. GitHub Pages allows us to serve static websites directly from our repositories, making it easy to share our work with others. A branch is just a separate version of the repository where we can make changes without affecting the main codebase. This is useful for testing new features or making updates without disrupting the live site. In my case, I just stuck to the "main" branch since I wasn't making any major changes that needed to be tested first. A commit is a snapshot of the changes made to the codebase at a specific point in time. Each commit has a unique identifier and a message that describes the changes made. This is useful when needing to look at a specific version of the code or when collaborating with others. Overall, using GitHub and GitHub Pages has been a great way to learn about version control and web development.
